# #15 — Replace spacing.scss JS Generator with Native SASS

> **Issue:** https://github.com/elycruz/atomic-material-js/issues/15
> **Package:** `@atomic/ui` (`packages/ui-js`)

## Problem

The `spacing.scss` file is generated by a TypeScript script (`node_scripts/gen-css/spacing.ts`) that iterates over pixel values and writes CSS custom properties. This adds unnecessary build complexity — the same output can be achieved natively in SASS using an `@each` loop.

## Approach

Replace the JS generator with a SASS `@each` loop that produces identical CSS output. Remove the JS script and simplify the build pipeline.

## Detailed Steps

### 1. Rewrite `packages/ui-js/css/modules/spacing.scss`

Replace the generated file with native SASS:

```scss
/**
 * spacing.scss
 *
 * The library's curated spacing properties, represented as `rem` units.
 */
@use "sass:math";

$spacing-values: (0, 1, 2, 3, 4, 5, 6, 8, 9, 12, 16, 18, 24, 32, 36, 48, 64, 72, 96, 144);

:root {
  @each $px in $spacing-values {
    --ez-#{$px}px: #{math.div($px, 16)}rem;
  }
}
```

This produces the same 20 custom properties (`--ez-0px: 0rem` through `--ez-144px: 9rem`).

### 2. Remove `packages/ui-js/node_scripts/gen-css/spacing.ts`

Delete the JS generator script entirely.

### 3. Update `packages/ui-js/node_scripts/gen-css/index.ts`

Remove the `genSpacingCss` import and invocation. Since `spacing.ts` was the only generator called, the orchestrator becomes empty. Leave as a minimal stub for future generators:

```typescript
/**
 * gen-css orchestrator.
 *
 * Currently no generators are active. Add future generators here.
 */
await Promise.resolve();
console.log('\ngen-css completed successfully (no generators active)\n');
```

### 4. Update `packages/ui-js/package.json`

Remove `pnpm gen:css &&` from the `build:css` script since there are no active generators:

```diff
-    "build:css": "pnpm gen:css && node node_scripts/build/index.ts",
+    "build:css": "node node_scripts/build/index.ts",
```

Keep the `gen:css` script entry for future use (the other files in `gen-css/` like `gen-material-themes.ts` and `font-size-props.ts` may be wired in later).

### 5. Verify

- Run `pnpm build:css` — confirm identical CSS output.
- Run `pnpm lint` / `pnpm lintfix` — no lint errors.
- Run `pnpm build` — full build succeeds.
- Run `pnpm test` — all tests pass.
